# Templates & Template Collections

This document explains templates and template collections used by PromptGrid Studio.

## Overview

- **Template**: A JSON object that represents a saved grid layout, cell contents, metadata, and optionally metadata used to reproduce a particular prompt or layout.
- **Template Collection**: A JSON file containing multiple templates. The project ships with `CoT_templates_collection.json` as an example collection.

## Where to find/import/export

- Example collection: `CoT_templates_collection.json` in the project root.
- Export: Use the app's Export functionality (UI component `ExportTemplatesCollectionModal`) to download a collection as JSON.
- Import: Use the app's Import functionality (UI components `ImportTemplatesCollectionModal` / `ImportModal`) to upload a collection.

## Typical template fields

The app does not enforce a single fixed schema in code, but templates generally include the following fields:

- `id`: string — unique identifier for the template (optional but recommended).
- `name`: string — human-readable name.
- `description`: string — short notes about the template.
- `grid`: object — the grid state (rows, columns, cell definitions).
- `cells`: array — individual cell objects; each cell typically includes `row`, `col`, `content`, and `style`.
- `createdAt` / `updatedAt`: string (ISO date) — timestamps (optional).

Example minimal template JSON:

```json
{
  "id": "template-1",
  "name": "Two-column phrase grid",
  "description": "Simple two-column layout",
  "grid": { "rows": 2, "cols": 3 },
  "cells": [
    { "row": 0, "col": 0, "content": "Hello" },
    { "row": 0, "col": 1, "content": "World" }
  ]
}
```

## Import / Export behaviour

- When importing a collection, the UI will parse the JSON and validate basic structure. Invalid JSON will be rejected.
- Exported collections are plain JSON arrays or objects containing multiple templates—these are safe to share and re-import.

## How to add templates in the app

1. Create or modify the grid in the editor workspace.
2. Click the Save Template action (UI uses `SaveTemplateModal`) and provide a name and optional description.
3. The template will be stored in local state and can be exported as a collection.

## Tips for collections

- Keep templates minimal if you intend to share them publicly (avoid sensitive content).
- Use clear `name` and `description` fields so users can pick templates quickly.
- When combining templates into a collection, ensure that `id` values do not collide or are unique.

## Troubleshooting

- Invalid JSON on import: check that the file is UTF-8, that it parses with `JSON.parse`, and that it contains an array or object of templates.
- Missing fields: many fields are optional; if the UI doesn't show an expected layout, check the `grid` and `cells` arrays exist and contain numbers for row/col.
